# 개발단계 분류 결정 트리

아래 트리는 “정답 회계처리”가 아니라 개발단계의 일관된 임시 기준입니다.

## 1) 입금 성격 결정

1. 입금 계좌가 조합/신탁 계좌인가?
- 예: `출자금(capital)` 후보
- 아니오: 2번으로

2. 조합 외 계좌 입금인가?
- 예: `차입성(debt-like)` 후보
- 아니오: 3번으로

3. 권리증 거래/권리증 환급과 직접 연관인가?
- 예: `권리증/환급성 항목`으로 표시하고 정산 라인에 반영
- 아니오: 4번으로

4. 증빙이 불충분하거나 성격 불명확한가?
- 예: `보류(suspense)`로 등록
- 아니오: 성격 확정 후 등록

## 2) 증빙 요구(개발단계 최소)

1. 출자금: 가입/출자 근거 문서
2. 권리증/환급성: 권리증 + 입금 확인
3. 차입성: 차입 약정(금액/만기/조건)
4. 보류: 증빙 미확보 사유 메모

## 3) 환불 계산 적용

1. 권리증 기준 환불(예: 3,000만원) 합산
2. 프리미엄 인정액 별도 합산
3. 손실/매몰비용 반영
4. 최종 환불선(`final_refund`) 생성

## 4) 예외 처리

1. 중복 지급참조번호: 등록 차단
2. 과지급 입력: 등록 차단
3. 지급 후 상태 불일치: `상태 동기화`로 보정
4. 회원 미연결 케이스: 연결 이슈로 분리 추적

## 5) 재분류 규칙

1. 변경 전/후 분류를 모두 저장
2. 변경 사유를 텍스트로 필수 저장
3. 변경자/시간 자동 기록 (`audit_logs`)
4. 리포트에서는 “현재 분류”와 “변경 이력”을 함께 조회
