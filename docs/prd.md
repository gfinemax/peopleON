## 1. 배경 및 해결 과제 (Problem & Context)

### 1.1 "휘발되는 히스토리" (Context Loss)
* **문제:** 담당자 변경 시 조합원과의 과거 약속(환불 구두 합의 등)이나 갈등 상황이 유실됨.
* **현황:** 엑셀에는 납부액만 기록되어 있어, "지난달 팀장이 해결해주기로 했다"는 주장을 검증할 수 없음.
* **해결:** 통화, 미팅, 문자 등 모든 접점(Touchpoint)을 **타임라인**으로 기록하여 영구 보존.

### 1.2 "정적인 분류의 한계" (Static vs Dynamic)
* **문제:** 단순 '조합원' 명부만으로는 소송 위험이나 협조 성향 파악이 불가능.
* **해결:** 상담 기록 기반 AI 태깅(`#강성민원`, `#납부약정`)으로 회원을 동적으로 분류.

### 1.3 "비효율적 행정" (Operational Bottleneck)
* **문제:** 미납 안내 문자나 주소 라벨 출력을 위해 매번 엑셀을 가공해야 하는 비효율 발생.
* **해결:** 시스템 내에서 원클릭으로 **문자 발송** 및 **라벨 출력(PDF)** 기능 제공.

## 2. 핵심 기능 요구사항 (Functional Requirements)

### 2.1 [CRM] 통합 액티비티 타임라인
* **통화/미팅 로그:** 언제, 누구와, 무슨 대화를 했는지 기록 (모바일 음성→텍스트 입력 지원).
* **발송 이력:** 안내문, 내용증명 발송 이력 및 수신 여부 자동 통합.
* **히스토리 뷰:** 특정 조합원 클릭 시 **[가입 ~ 현재]** 모든 이력을 시간순으로 표시.

### 2.2 [AI] 이력 기반 자동 분류
* **자동 태깅:** 상담 내용에서 키워드 추출 (`#소송불사`, `#자금부족`) 후 태그 부착.
* **등급 조정:** "자금 융통 가능" 기록 시 → `[납부대기]` 등으로 상태 변경 제안.

### 2.3 [ERP] 분담금 자동 청구 및 수납
* **자동 청구:** `Tier(지주/일반)` × `Unit(평형)`에 따른 분담금 자동 계산.
* **입금 매칭:** 은행 엑셀 업로드 시 입금자명/동호수 분석 후 자동 수납 처리.
* **미납 관리:** 회차별 미납자 리스트 실시간 추출 및 독촉 문자 연동.

### 2.4 [Ops] 행정 자동화
* **문자 발송:** 타겟 그룹(예: 3회차 미납자)에게 대량 문자 발송 및 이력 자동 저장.
* **라벨 출력:** 등기/실거주 주소 선택 후 라벨지(폼텍 등) 규격 PDF 자동 생성.

### 2.5 [Interface] 하이브리드 입력 시스템
* **📱 Mobile (현장 최적화 + Native App):**
    *   **Dashboard:** 전체 조합원 수, 납부율 등 ERP/CRM 핵심 지표 실시간 연동.
    *   **Finance View:** 개인별/회차별 납부 현황 및 미납 내역 실시간 조회 (진행률 그래프 포함).
    *   **Finance View:** 개인별/회차별 납부 현황 및 미납 내역 실시간 조회 (진행률 그래프 포함).
    *   **Quick Log:** 현장에서 즉시 통화/미팅 기록 및 음성 메모 지원.
    *   **Mobile Search:** 조합원/동호수 즉시 검색 (전체 화면 팝업).
* **💻 PC (관리 최적화):** 상세 상담 일지 작성, 녹취/문서 파일 첨부, 대량 데이터 처리.

### 2.6 [UI/UX] 상세 정보 시각화 고도화
*   **폴더 탭 (Folder Tabs):** 탭과 콘텐츠 배경이 연결되는 일체형 디자인으로 몰입감 제공 (조명 효과 포함).
*   **입력 편의성:** 상담 내역 입력 시 '음성 인식' 아이콘 제공 및 직관적인 작성 창 배치.
*   **가독성 개선:** 납부 현황 테이블의 불필요한 공백 제거 및 용어 순화 ('수납완료' -> '완료').
*   **UI 일관성 및 편의성 개선:** 
    * 管理者 관리자 메모상시 노출로 중요 정보 누락 방지.
    * 감사 로그(Audit Logs) 행위자/대상 병합 및 상세 내용 요약 표시로 시스템 모니터링 가독성 극대화.
*   **데이터 정합성 강화:** 
    * 실제 권리증 리스트(`asset_rights`)를 기준으로 오래된 엑셀 임포트 데이터 무시 및 중복/고스트 데이터 표시 방지.
    * 대시보드에서 권리증 충돌(서로 다른 조합원이 동일 권리증 보유) 감지 시 '조치 필요 조합원' 최상단에 그룹화 및 하이라이트 (🚨 펄스 뱃지) 제공.
    * **권리증 통합 검수 및 분석:** `person_certificate_summaries` 기반으로 등기/기타 조합원의 권리증 소유 상태(잠정/최종, 충돌, 수동고정 등)를 Finance 및 Settlement 대시보드에서 실시간 집계. 검색 및 리스트 뷰에서 병합된 권리증 번호를 일관되게 표시.
    * **대리인 권리증 합산:** 등기조합원에 연결된 대리인(agent)의 권리증을 본인 데이터에 자동 합산하여 리스트, KPI 차트, 엑셀, 인쇄에 모두 반영. 중복 카운팅은 고유 번호 기준(`Set`)으로 방지.

### 2.7 [Ops] 맞춤형 데이터 출력
*   **엑셀 다운로드(컬럼 선택형):** 필터링된 인물 목록을 대상으로, 사용자가 포함할 컬럼(성명, 연락처, 권리증번호, 구분, 동호수, 주소, 상태, 역할, 메모 등)을 직접 선택하여 `.xlsx` 엑셀 파일로 즉시 다운로드. 권리증번호는 화면 축약(`외 N건`) 없이 전체 번호가 온전히 출력됨.
*   **A4 보고서 인쇄:** 브라우저 인쇄 기능(`Ctrl+P`)을 통해 A4 Landscape 규격에 최적화된 깔끔한 테이블 형태로 인쇄 또는 PDF 저장 가능.

## 4. 데이터 정합성 강화 (Data Integrity)
- 발생 가능한 **동명이인 중복 데이터(Ghost data)** 및 **번호 오기입(예: 생년월일이 회원번호에 기입됨)**을 로직 및 스크립트로 정제하여, 사용자 단에서 깔끔하고 유일한 조합원 뷰를 제공합니다.
- 리스트 뷰와 상세 뷰의 조합원 식별 로직을 동기화하여 시각적 불일치를 제거했습니다.
